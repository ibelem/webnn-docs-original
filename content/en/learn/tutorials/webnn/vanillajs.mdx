# Tools for Generating WebNN Vanilla JavaScript

Package-size sensitive applications require minimal binary footprints to reduce download overhead. 

Vanilla JavaScript provides the smallest bundle size for building WebNN graphs and performing computations, but requires more development effort than higher-level libraries like Transformers.js and ONNX Runtime. 

To address this trade-off, the WebNN community is developing tools that generate optimized WebNN JavaScript code from existing models, balancing performance with bundle size requirements.
