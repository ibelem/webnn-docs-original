## How do I file an issue with WebNN?

File issues to following WebNN issue trackers:

- [Design or Specification](https://github.com/webmachinelearning/webnn/issues)
- [Chromium Implementation (Component: Chromium > Blink > WebML)](https://issues.chromium.org/issues?q=status:open%20componentid:1456206&s=created_time:desc)
- [ONNX Runtime Web or WebNN Execution Provider](https://github.com/microsoft/onnxruntime/issues)
- [WebNN Samples](https://github.com/webmachinelearning/webnn-samples/issues)
- [WebNN Developer Preview Demo](https://github.com/microsoft/webnn-developer-preview/issues)

If you're unsure about which issue type to select, feel free to submit your issue [here](https://github.com/webmachinelearning/webnn-samples/issues) and our team will help categorize and resolve it.

## How do I debug issues with WebNN?

The WebNN API uses standard DOM exceptions for error propagation. When an error occurs, the API throws specific exceptions that can be caught and handled in your code.

Check the GPU information log, these pages contain detailed diagnostic information about GPU-related errors, histograms and performance.

```javascript
about://gpu
about://histograms/webnn
```

### Additional Resources

- Review the [W3C WebNN Specification](https://www.w3.org/TR/webnn/) for complete error handling documentation

## What operating systems does WebNN support?

WebNN currently supports Windows, Android, ChromeOS, and macOS. The availability of WebNN operations may vary depending on the device type (CPU, GPU, NPU) and operating system. Progress is ongoing, so your patience is appreciated.
